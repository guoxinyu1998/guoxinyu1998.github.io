Finding Task-Relevant Features for Few-Shot Learning by Category Traversal

Abstract

作者认为现有的基于metric的方法都独立地对待支持集，没有从整体上来看一个meta-task，因此提出了**（Category Traversal Module）CTM模块**，可以直接嵌入任何metric的方法，这样一个模块可以一次性遍历所有的支持集图像，基于类内共性和类间唯一性来找到和task相关的特征。

Introduction

FSL介绍略

解决FSL问题的有效方法是训练一个神经网络来产生特征嵌入，用于将查询样本与每个支持样本进行比较。通过从更大的标记训练集中迭代地采样查询和支持集，不断优化来最大化查询样本和相同标记样本之间的相似性得分，来为FSL分类问题进行显式训练。如图1（a）所示：

fθ(⋅)f_{\theta}(\cdot)fθ​(⋅)是一个嵌入函数，简单来说就是，（a）中的查询样本qqq和support set中每个类的样本进行单独比较，忽略了类之间的相关信息，没有从整体上来看一个meta-task；而（b）中提出的CTM遍历支持集中的类别来寻找任务相关的特征，也就是将support set看成了一个整体。

本文扩展了基于度量学习的方法，将整个支持集作为一个整体来考虑，模型可以找到与每个任务最相关的维度。这在FSL中尤其重要：因为每项任务只有很少的标记数据，所以必须充分利用所有样本中的信息。

举两个例子说明这样做的意义：

图2

对于上图来说，这是一个分类任务。具体步骤还是和metric learning一样，计算query 与support中每一个的距离，最后得到结果。

  • （a）是5-way 1-shot的小样本学习任务，其中只有两个简单的特征维度：颜色和形状，最终的目标是确定query属于支持集中的哪个类。可以看到query是一个绿色的圆形，由于支持集中每个类的样本的颜色都是不同的，而有些样本的形状是相同的，所以在该任务中最相关的特征就是颜色，因此query所属的类应该是（iii）。但是如果在计算query的特征相似性时没有考虑整个支持集，而是一个类一个类单独地看，那么query与类（i）（iii）（iv）（v）就会有相同的相似度，准确率就会降低。只有在考虑整个支持集的上下文信息的情况下，我们才发现颜色是最相关的特征。这个例子就启发我们要遍历支持集中的所有类别以找到具有类间唯一性（inter-class uniqueness）的特征。
  • （b）是具有multi-shot的小样本学习任务，可以看到在类（ii）中，大多数样本的颜色都是相同的，而它们的形状各有不同，因此在类（ii）中，颜色是最相关的特征，也即具有类内通用性（intra-class commonality）的特征。

由上面两个例子可以看出，纵观整个support set是非常有必要的，利用类间唯一性和类内通用性，可以找出与某任务最相关的特征。在一个类中，通过对特征向量取平均，可以缓解类内样本间的不同，从而得到类内的共享特征，即具有类内通用性的特征。

Algorithm

FSL

在一个FSL分类任务中，得到一个由NNN个不同的，以前未见过的类别组成的支持集，每个类有K个样本。给定一个查询样本，目标是将其分类到NNN个支持类别之一。

Training

使用带标签样本的大型数据集CtrainC^{train}Ctrain训练一个模型（其中的类别和用于测试的类别标签不同），使用episodes进行训练，每个episode中，构建一个support set SSS和一个query set QQQ：

Inference

泛化性能是在测试集集上测试的，SSS，QQQ是从CtestC_{test}Ctest​中采样的，CtestC_{test}Ctest​中的类别与CtrainC_{train}Ctrain​不同，支持集中的标签是已知的，而查询中的标签是未知的，并且仅用于评估。

Category Traversal Module (CTM)

CTM主要由两部分组成：

  • concentrator unit，用于提取类内具有通用性的embedding；
  • projector unit，通过考虑concentrator unit的输出，得到类间具有唯一性的embedding。

Concentrator: Intra-class Commonality

concentrator的目标是找到一个类中的所有样本共享的通用特征，也即具有类内通用性的特征。将特征提取器的输出为(NK,m1,d1,d1)(NK,m_{1},d_{1},d_{1})(NK,m1​,d1​,d1​)，其中m1m_{1}m1​表示通道的数量，d1d_{1}d1​表示空间大小，那么concentrator可以被定义为：

fθ(S):(NK,m1,d1,d1)⟶ Concentrator o:(N,m2,d2,d2)f_{\theta}(\mathcal{S}):\left(N K, m_{1}, d_{1}, d_{1}\right) \stackrel{\text { Concentrator }}{\longrightarrow} \boldsymbol{o}:\left(N, m_{2}, d_{2}, d_{2}\right) fθ​(S):(NK,m1​,d1​,d1​)⟶ Concentrator ​o:(N,m2​,d2​,d2​)

其中m2m_{2}m2​示输出的通道数量，d2d_{2}d2​表示输出的空间大小。其实输入首先被送入一个CNN模块中进行降维，然后对每个类内的样本取平均，从而获得最终输出ooo。在1-shot情况下就不用进行平均操作了，因为每个类中只有一个样本。

这个CNN模块可以是一个简单的CNN层，也可以是一个ResNet块，降维的目的是移除样本之间的差异，从而能够在类内提取出具有通用性的特征。也就是说，从d1d_{1}d1​到d2d_{2}d2​就是一种降维，也就是适当程度的下采样，在经过降维之后，再对类内样本取平均，以得到最终输出ooo。实验证明这种方法比原型网络中直接对样本取平均的方法要好，而当m1=m2,d1=d2m_{1} = m_{2},d_{1} = d_{2}m1​=m2​,d1​=d2​ 时，原型网络中的取平均方案也可以看作是concentrator的一种特殊情况。

简单来说就是先降维，再取平均，得到类表示，看下图一目了然：

Projector: Inter-class Uniqueness

projector的目的是**屏蔽掉其它不相关的特征，然后选择一个对于当前任务来说最具有判别性的特征。**在给定由concentrator输出的特征的情况下，projector同时考虑支持集中所有类别的concentrator特征，然后得到输出ppp：

o^:(1,Nm2,d2,d2)⟶ Projector p:(1,m3,d3,d3)\hat{\boldsymbol{o}}:\left(1, N m_{2}, d_{2}, d_{2}\right) \stackrel{\text { Projector }}{\longrightarrow} p:\left(1, m_{3}, d_{3}, d_{3}\right) o^:(1,Nm2​,d2​,d2​)⟶ Projector ​p:(1,m3​,d3​,d3​)

其中o^\hat oo^是ooo的一个reshaped version，m3,d3m_{3},d_{3}m3​,d3​分别是输出的通道数量和空间大小。为了实现跨类遍历的目标，首先将NNN中的类别原型连结到m2m_{2}m2​，然后在连结后的特征上应用一个CNN来生成一个大小为(1,m3,d3,d3)(1,m_{3},d_{3},d_{3})(1,m3​,d3​,d3​)的映射，最终在m3m_{3}m3​上应用softmax来生成一个maskppp，这个ppp用于对与该任务最相关的特征维度进行mask。
看图：

过程主要分为三步：

  • 首先，对Concentrator模块提取到的feature进行拼接
  • 然后在拼接后的向量上使用卷积层进行卷积，来拟合不同类别的信息
  • 最后在所得到的feature map的last-channel上进行softmax操作得到最终的向量
Reshaper

为了使用这一模块得到的向量，需要将support set和query set得到的向量进行conv-reshape操作：

改进的特征III的生成

在concentrator和projector的支持下，经过跨类遍历可以生成一个mask：fθ(S)→pf_{\theta}(S) \rightarrow pfθ​(S)→p，然后要将这个ppp与支持集和查询集的feature embedding结合起来，生成改进的特征I(⋅)I(\cdot)I(⋅)，才能实现CTM的效果。而这种结合，也就是I(⋅)I(\cdot)I(⋅)的生成方式，在支持集和查询集中是不同的。

在查询集中，因为query没有类别标签，因此这种结合只是对embedding和ppp进行简单的元素乘法（element-wise multiplication）：I(Q)=r(Q)⨀pI(Q) = r(Q)⨀pI(Q)=r(Q)⨀p；

在支持集中，由于我们知道query的类别标签，因此要么直接将ppp mask到embedding上，要么保持(m2,d2,d2)=(m3,d3,d3)(m_{2},d_{2},d_{2})= (m_{3},d_{3},d_{3})(m2​,d2​,d2​)=(m3​,d3​,d3​)，然后用ppp对concentrator的输出ooo进行mask：

option1:I1(S)=r(S)⊙p:(NK,m3,d3,d3),option2:I2(S)=o⊙p:(N,m3,d3,d3).option 1: \mathcal{I}^{1}(\mathcal{S})=\boldsymbol{r}(\mathcal{S}) \odot p:\left(N K, m_{3}, d_{3}, d_{3}\right),\\ option 2: \mathcal{I}^{2}(\mathcal{S})=\boldsymbol{o} \odot p:\left(N, m_{3}, d_{3}, d_{3}\right) . option1:I1(S)=r(S)⊙p:(NK,m3​,d3​,d3​),option2:I2(S)=o⊙p:(N,m3​,d3​,d3​).

实验证明，option1的效果更好。
