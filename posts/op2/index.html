Label Confusion Learning to Enhance Text Classification Models

Ａbstract

one-hot表示不能充分反映样本和标签之间的关系，标签之间通常不是完全独立的，并且样本可能与多个标签相关。在对标签信息极为相似的数据集进行预测时，one-hot表示可能会带来过拟合的问题。

本文提出了标签混淆模型LCM，可以在训练过程中通过计算样本和标签之间的相似度，得到一个反映标签之间的混淆/相似程度的分布，最后，使用该混淆分布来调整原来的one-hot分布，提高分类性能。

Introduction

文本分类模型通用学习模式

文本分类的模型有一个通用的学习模式：

  1. 一个用于文本表示的深度模型(CNN 、RNN、BERT、LSTM)
  2. 一个简单的分类器对标签进行预测(Softmax)
  3. 运用交叉熵损失函数计算真实标签（one-hot表示）和预测概率分布间的损失，从而进行优化
one-hot表示存在的两个问题
  • one-hot标签表示是基于所有类别独立的假设，但是在实际场景中，标签通常不是完全独立的，并且样本可能与多个标签相关，尤其是对于标签相似的数据集。one-hot表示没有考虑样本和标签之间的联系，限制了模型的学习的能力。
  • 容易受到错误标记样本的影响

举个例子：

六个类别的分类任务，利用交叉熵损失计算loss：

loss：$l = - \sum{i}^{k}y{i}log(p_{i})$

可以看到，只有$y=1$那一维度参与了loss的计算，其他的都忽略掉了，带来的后果：

  • 真实标签跟其他标签之间的关系被忽略了，很多有用的知识无法学到；比如：“鸟”和“飞机”本来也比较像，因此如果模型预测觉得二者更接近，那么应该给予更小的loss
  • 倾向于让模型更加“武断”，成为一个“非黑即白”的模型，导致泛化性能差
  • 面对易混淆的分类任务、有噪音的数据集时，更容易受影响
本文的贡献
  • 构造了一个插件——“Label Confusion Model(LCM)”，可以在模型训练的时候实时计算样本和标签间的关系，从而生成一个标签分布，作为训练的target，实验证明，这个新的target比one-hot标签更好
  • 这个插件不需要任何外部的知识，也仅仅在训练的时候才需要，不会增加模型预测时的时间，不改变原模型的结构。所以LCM的应用范围很广
  • 实验发现LCM还具有出色的抗噪性和抗干扰能力

Related work

一些解决的办法：

  • LDL
  • Label Enhancement
  • Loss Correction

Method

构思：

一个合理的标签分布应该是什么样的
  • 标签分布应该可以反映标签之间的相似性，比如“鸟”和“飞机”的相似度更高（都有翅膀，都可以飞），“鸟”和“汽车”的相似度更低
  • 标签间的相似性是相对的，同样的标签，对于不同的句子，标签之间的相似度也是不一样的，比如句子中的语句为“……在空中飞……”，那”飞机“和“鸟”的相似度就很高，当句子内容设计到“……速度快的交通工具时……”，“飞机”和“汽车”的相似度就很高。
  • 构造得到的标签分布，在01化之后应该跟原one-hot表示相同，就是构造出的标签分布，最大值对应的标签不能跟原本的one-hot标签不一致，最终的标签分布，还是要以one-hot为标杆来构造

因此可以利用样本和标签之间的关系来构造标签分布，从而对于具有相同标签的不同样本，标签分布将动态地改变。对于文本分类问题，可以通过计算文本表示和每个标签之间的相似性来模拟标签分布。这样，不仅捕获了样本和标签之间的关系，标签之间的依赖关系也通过这些关系得到了体现。基于这一基本思想设计了一个标签混淆模型LCM，通过计算样本和标签之间的语义关系来学习模拟标签分布（SLD）。然后，SLD被视为真实的标签分布，并与预测的分布进行比较，通过KL-散度计算损失。

模型：

LCM-based Classification Model

  • 编码器，如CNN、RNN、BERT
  • 简单的分类器：softmax
  • 得到文本表示
  • softmax对文本表示进行预测，得到预测标签分布：

其中$f$是输入编码器函数，把长度为$n$输入序列$x = [x{1},x{2},x{3},…,x{n}]$转换为长度为$n$，维度为$d$文的文本表示$v^{(i)}= [v{1}^{(i)},…,v{n}^{(i)}]$，$y^{(p)}$是预测的标签分布PLD

  • 标签编码器：深度神经网络，对标签进行编码，得到标签表示矩阵
  • 模拟标签分布(SLD)计算模块：
    • 相似度计算层：以标签表示和当前样本表示为输入，通过点积计算它们的相似度值，然后应用softmax激活的神经网络得到标签混淆分布（LCD）。LCD通过计算实例和标签之间的相似度来捕捉标签之间的相关性。因此，LCD是动态的、实例相关的分布。
      • Q：计算点积涉及到维度的问题，样本的维度和标签表示的维度是多少？
      • Q：用一个softmax是什么意思？
      • A：对得到的相似度向量进行规范化，使LCD相加为1（使样本在标签上的分布相加为1）
    • SLD计算层：通过一个可调控的参数$\alpha$来将LCD加到one-hot表示中，softmax进行归一化，生成SLD，控制参数$\alpha$决定了改变one-hot向量的程度。
      • Q：此处的softmax是何意？
      • A：相加之后，样本在标签上的分布之和不再为1，需要进行规范化

计算过程：

$f^{L}$是标签编码器函数，把标签$l = [l{1},l{2},…,l_{C}]$转变为表示矩阵$V^{(l)}$，$C$是类别数量，注意，输入到LCM的标签序列的顺序应该与one-hot目标相同。$y^{(c)}$是LCD，$y^{(s)}$是SLD。

然后，SLD被视为新的训练目标，以取代one-hot向量并监督模型训练，使用KL散度作为损失函数计算模拟标签分布SLD$y^{(s)}$和预测的标签分布$y^{(p)}$间的差异：

本文主要是针对标签的分布以及标签之间的关系进行细致的建模，在考虑输入的情况下分析标签之间的依赖关系，从而实现动态的输入感知的标签编码，进而帮助模型充分利用标签数据。

Experiments

Choice of Basic Predictors

  • LSTM

  • CNN

  • BERT

  • Settings

  • LSTM:嵌入大小和隐藏大小为64

  • CNN：使用大小为3、10、25的滤波器（三个），每个卷积块的滤波器数量为100个

详细参数设置

对于LSTM和CNN，如果不使用预先训练的单词嵌入，嵌入大小为64。否则中文任务嵌入大小为250，英文任务为100。中文单词嵌入是使用skip-gram算法在1GB左右的中文维基百科语料库中预先训练的。英文单词嵌入是GloVe嵌入

BERT：从BERT模型中获得文本表示，然后将文本表示的维数减少到64

本文中的LCM组件是由embedding lookup layer 和dense neural net组成的，嵌入大小和隐藏大小保持与basic predictor相同（维度相同，计算点积相似度）

Adam进行优化，学习率：0.001，batch：128 Keras实现

embedding lookup layer：将标签映射为低维稠密向量

dense neural net：稠密卷积神经网络

Results

Test performance

实验结果见原文

结果表明，LCM具有增强文本分类模型性能的能力。分类模型加了LCM后，效果比基线都有不同程度的提高……

以20NG数据集为例，说明LCM的作用：

20NG数据集中包含20个类别，但是这些类别可以分为几组，同一组中的标签很相近，难以区分，对20个标签的表示进行了可视化（t-SNE），如下图所示，标签表示是从LCM的嵌入层中提取的，下图a是标签表示的余弦相似性矩阵，其中对角线之外的元素反映了一个标签与另一个标签的相似程度。可视化如图b所示，可以发现容易混淆的标签，尤其是同一组的标签，标签表示相近，由于标签表示在开始时都是随机初始化的，我们可以看到LCM能够学习反映标签间混淆关系的标签表示。

LCM起作用的原因
  1. LCM在训练过程中学习模拟标签分布SLD，通过考虑输入样本和标签之间的语义相似性来捕捉标签之间的复杂关系。这优于简单地使用one-hot向量
  2. 对于具有大量类别或标签非常相似的数据集，往往存在错误标注的样本。在这种情况下，使用one-hot标签表示的训练往往会受到这些错误标注数据的影响。然而，使用SLD，错误标签的索引上的值将被削弱并分配给那些相似的标签。因此，错误标签的影响是相对微不足道的。
  3. 当给定的标签具有某种相似性时(例如，“计算机”和“电子”在语义上是相似的主题，在内容上共享许多关键词)，用标签分布来标注文本样本是自然合理的，标签分布包含的信息更充分。然而，目前的分类学习范式忽略了样本之间的差异，导致这一重要信息丢失。

The Effect of α and Early Stop of LCM

在生成SLD时，$\alpha$越大，原始one-hot标签的权重就越大，为了确保SLD中的最大值与原始one-hot标签中的最大值保持相同的位置，$\alpha$最少为0.5

基于LCM的模型比基线模型学习速度快得多，较小的$\alpha$模型收敛速度更快，但是当$\alpha$过小时，如0.5，模型将极容易过拟合，这种情况下可以应用early-stop策略。从$\alpha$为0.5的曲线（下图a）上可以看出，在10个epoch后，准确率开始下降，可以在大约10次迭代时停止LCM。下图b所示的结果展示了early-stop的有效性，即可以防止模型过拟合。α和early-stopping策略的选择应基于特定数据集和任务的性质。根据我们的实验经验，在具有相似标签或标签错误的数据集上，应该选取较小的$\alpha$。

数据集混淆度的影响

构造了几个“简单的”和“困难的”数据集，通过实验标签，LCM更适合那些容易混淆的数据集
