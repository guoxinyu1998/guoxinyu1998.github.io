github+typora+picgo优雅解决博客中的图片问题

发表于2022-03-22|更新于2022-03-23|✍️博客撰写
|字数总计:1.6k|阅读时长:5分钟|阅读量:

介绍

要想在博客里插入图片，一般来说有两种方法：

  1. Hexo自带的post_asset_folder功能
  2. 搭建图床

下面就这两种方法进行一些简单说明

post_asset_folder

将项目站点文件config.yml中的post_asset_folder设置为true后，通过hexo n "xxx.md"创建新的博客文件时，会在_post目录下生成相同名字的文件夹。可以将图片放在该文件夹中，并在.md文件中以相对路径的方式进行访问。

先来说说缺点吧，因为缺点实在是太明显了……

  • 当创建的.md文件过多时，整个_post文件夹会显得十分臃肿，而别并不是每个博文都需要图片或者只需要一两张图，大量不必要的文件夹对博客的编译和上传造成了很大的负担。
  • 访问速度慢，因为github服务器在国外，而通过相对路径方式调用的图片无法实现CDN加速
  • 最让人头疼的引用问题来了。假设_post下有一篇名为1.md的博文，对应的有一个文件夹1，里面有一张需要的图片xxx.jpg，hexo要求我们在.md中以![](xxx.jpg)的格式引用，而markdown中引用的格式是![](1/xxx.jpg)。若我们以![](xxx.jpg)的格式引用，网站的博客上可以显示图片，而本地.md文件中不能显示；如果以![](1/xxx.jpg)的格式引用，本地可以显示，而网站博客不能显示……

实在想不到有啥优点……

搭建图床

说白了就是将图片上传到某个可以生成外链的地方，在.md中通过外链进行访问

常用的图床有：

  • 阿里云/腾讯云/七牛云的对象存储
  • 微博图床
  • github

本文将详细介绍github图床的搭建方法以及对应的CDN加速，并结合typora与PicGo在博客撰写过程中实现快速上传。

github图床既免费又好用，而且每个用户有100G的个人空间（真是良心啊）！

不过切记要时刻注意github仓库的使用情况，千万不能超过1GB！

如果超过了1GB，恭喜你，你的仓库将会受到github的制裁，同时还会面临官方的文件查验，如果被发现你的账号利用github做私人图床，那么再次恭喜你，可以考虑换个新的账号了……

  • 提高网站访问速度
  • 本地访问友好
  • 减轻hexo的编译和上传压力

要非说有啥缺点吧，就是得注意单个仓库不要超过1GB

对于Mb，甚至十几Mb的图片，需要压缩之后再上传，不然这么大的图片github仓库吃不消，而且访问起来也费劲，具体的压缩网站将会在下面的步骤中详细说明！

好了，废话不多说了，开启github图床的搭建之旅吧

具体方法

实际使用分为两种情况

  • 本地文件夹上传：网站封面图，背景图等等
  • typora上传：博文中需要的图片

  1. 右上角+号—创建github public仓库，注意一定是公有，其他内容自行设置即可

  2. 图片压缩工具

    name tinypng inagine
    压缩质量 无损压缩 有损压缩
    工具类型 在线网站 .apk
    说明 每天最多20张图，每张限制为5Mb 无限制，支持同时处理多张图片

这两个都能达到有效压缩图片的目的，且肉眼几乎无法分辨出压缩前后的图片。使用哪个视个人需求而定吧。

  1. 进入任意目录下，通过git命令将仓库传输到本地

    直接下载仓库也行。

  2. 图片压缩后放到本地仓库，通过git bash上传

  3. CDN加速

    使用 jsdelivr 提供的免费 CDN 服务，无需注册，方便有效！
