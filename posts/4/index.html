Learning to Compare Relation Network for Few-Shot Learning

Learning to Compare : Relation Network for Few-Shot Learning

2018 CVPR会议论文

论文解读

Abstract

关系网络

在特征提取和度量方式上都是端到端的训练

在元学习过程中，学会学习深度距离度量方式，在episode中进行少量图像的比较

一旦经过训练，RN能够通过计算查询图像和每个新类别的几个例子之间的关系得分来对新类别的图像进行分类，而无需进一步更新网络。

1.Introduction

具体来说，提出了一个具有两个分支的关系网络(RN)，它通过学习将查询图像与few-shot标记的样本图像进行比较来执行few-shot识别。首先，嵌入模块生成查询和训练图像的表示。然后由关系模块比较这些嵌入，确定它们是否是相同的类。

采用episode的训练方式

嵌入模块和关系模块是端到端元学习的

可学习的非线性比较器，而不是一个固定的线性比较器

我们的方法优于以前的方法，同时更简单(没有RNNs [1，2，3])和更快(没有微调[3，4])。

2.Related Work

Learning to Fine-Tune

成功的MAML方法[1]旨在元学习一个初始条件(一组神经网络权重)，该条件有利于对few-shot问题进行微调。这里的策略是搜索给定神经网络的权重配置，使得它可以在几个梯度下降更新步骤内有效地微调稀疏数据问题。许多不同的目标问题是从一个多任务训练集中抽取的；然后对基本神经网络模型进行微调，以解决每个问题，微调后每个目标问题的成功会推动基本模型的更新，从而推动容易微调的初始条件的产生。few-shot优化方法[2]在元学习方面更进了一步，不仅获得了良好的初始条件，而且获得了经过训练的基于LSTM的优化器，该优化器在微调方面特别有效。然而，这两种方法都需要对目标问题进行微调。相比之下，我们的方法以完全前馈的方式解决目标问题，不需要模型的更新。

RNN Memory Based

另一类方法利用带有记忆的递归神经网络[3，4]。这里的想法是典型的RNN迭代给定问题的样本，并在其隐藏的激活或外部记忆中积累解决该问题所需的知识。新的样例可以被分类，例如通过将它们与存储在存储器中的历史信息进行比较。因此，“学习”一个单一的目标问题可能发生在展开RNN的过程中，而“学会学习”意味着通过学习许多不同的问题来训练RNN的权重。尽管这些体系结构很有吸引力，但它们面临的问题是确保它们可靠地存储所有潜在的长期历史相关信息，而不会忘记。在我们的方法中，我们避免了重复出现的网络的复杂性，以及在确保它们的存储足够性时所涉及的问题。相反，我们的“学会学习”方法完全由简单快速的前馈神经网络定义。

Embedding and Metric Learning Approaches

当学习目标few-shot问题时，先前的方法具有一定的复杂性。另一类方法旨在学习一组投影函数，这些函数从目标问题中获取查询和样本图像，并以前馈方式对它们进行分类[5，6，7]。一种方法是根据样本集参数化前馈分类器的权重[7]。这里的元学习是训练辅助参数化网络，该网络学习如何根据少量样本集来参数化给定的前馈分类问题。基于度量学习的方法旨在学习一组投影函数，以便在嵌入中表示时，使用简单的最近邻或线性分类器很容易识别图像[5，6，8]。在这种情况下，元学习的可转移知识是投影函数，目标问题是简单的前馈计算。

与我们的方法最相关的是[6]的原型网络和[8]的siamese网络。这些方法侧重于学习嵌入，这些嵌入对数据进行变换，从而可以用固定的最近邻[6]或线性[8，6]分类器进行识别。相比之下，我们的框架以[9，10，11]的方式进一步定义了一个关系分类器CNN(而[33]侧重于推理同一图像中两个对象之间的关系，以解决不同的问题。).与[8，6]相比，这可以被视为提供了一个可学习的而不是固定的度量，或者非线性而不是线性的分类器。与[8]相比，我们受益于从头开始的端到端方式的episode训练策略，与[4]相比，我们避免了样本集的 set-to-set RNN嵌入的复杂性，并且简单地依赖于pooling[9]。

3.方法

3.1问题定义

有三个数据集:一个训练集、一个支持集和一个测试集。支持集和测试集共享相同的标签空间，但训练集有自己的标签空间，与支持/测试集不相交。如果支持集包含每个C类的K个标记的例子，目标few-shot问题称为C-way K-shot。

仅使用支持集，原则上我们可以训练一个分类器，为测试集中的每个样本$\hat x$分配一个类标签$\hat y$。然而，由于支持集中标记样本很少，这种分类器的性能通常不令人满意。因此，我们的目标是在训练集上执行元学习，以便提取可转移的知识，这将允许我们在支持集上执行更好的few-shot学习，从而更成功地对测试集进行分类。

开发训练集的一个有效方法是通过基于episode的训练来模仿少few-shot learning设置，如[39]中所建议的。在每一次训练迭代中，通过从训练集中随机选择C类，其中来自每个C类的K个标记样本用作 sample set $S = \{(x_{i},y_{i})\}_{i=1}^{m}(m = K \times C)$,C类样本的剩余部分的一部分用作query set $Q = \{(x_{j},y_{j}) \}_{j=1}^{n}$。设计样本/查询集来模拟在测试时的支持/测试集。如果需要，可以使用支持集进一步微调从样本/查询集训练的模型。在这项工作中，我们采用了这样一种基于episode的培训策略。在我们的few-shot实验中(见第4.1节)，我们考虑了one-shot(K = 1，图1)和five-shot(K = 5)设置。我们还解决了第3.3节中解释的K = 0 zero-shot学习的情况。

3.4 网络结构

由于大多数FSL模型利用四个卷积块嵌入模块，我们遵循相同的架构设置进行公平比较，见图2。更具体地说，每个卷积块分别包含一个64个3 × 3大小的过滤器、一个批规范化和一个ReLU非线性层。前两个块还包含一个2 × 2最大池层，而后两个块不包含。我们这样做是因为我们需要关系模块中进一步卷积的输出特征映射。关系模块由两个卷积块和两个全连接层组成。每个卷积块是有64个3 x 3大小的滤波器，随后是批规范化、ReLU非线性和2 × 2max-pooling。最后一个max-pooling层的输出大小，对于Omniglot和miniImageNet分别为$H = 64$和$H = 64 \ast 3 \ast 3$。两个完全连接的层分别是8维和1维的。除了输出层是Sigmoid之外，所有完全连接的层都是ReLU[1]，以便为我们网络架构的所有版本生成合理范围内的关系分数

5.为什么RN可以工作？

5.1与现有模型的关系

先前工作使用固定的预先指定的距离度量，例如欧几里德或余弦距离来执行分类[39，36]。所有的学习都发生在特征嵌入中，使用固定的度量。

关系网络可以被视为既学习深度嵌入又学习深度非线性度量(相似性函数)，这些都是端到端的相互调整

以前的固定度量方法严重依赖于学习的嵌入网络的效率，受到嵌入网络生成的区别性表示不够充分的限制。

通过学习到的相似性矩阵比人为选定的矩阵更具有灵活性，更能捕捉到特征之间的相似性。
