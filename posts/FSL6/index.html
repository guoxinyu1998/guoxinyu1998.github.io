Diverse few-shot Text classification with multiple metrics

Abstract

与许多现有的基于度量或优化的元学习在图像领域的工作相比，本文考虑了一个更现实的环境，其中任务是多样的。

然而，它给现有的最先进的基于度量的算法带来了巨大的困难，因为单个度量不足以捕捉自然语言领域的复杂任务变化，本文提出了一种自适应的度量学习方法。

从元训练任务中获得一组度量，再用这组度量自动确定最佳加权组合，用于新的FSL任务。

Introduction

以前的FSL主要考虑图像领域，所有任务通常从一个大的数据集中采样，使任务来自同一个领域，几乎以前所有的工作都采用一个通用的元模型来处理所有的FSL任务。

但这种设置与FSL中文本分类的现实应用场景相去甚远。例：企业人工智能云服务上，许多客户端提交各种任务来为特定业务目的训练文本分类模型。这些任务可以是对客户对不同产品/服务的评论或意见进行分类，监控公众对不同政策变化的反应，或者确定用户对不同类型的个人助理服务的意图。由于大多数客户无法收集足够的数据，他们提交的任务形成了一个FSL问题。此外，这些任务差异很大，因此一个通用的度量标准不足以处理所有这些任务。

我们考虑一个更现实的FLS setting，任务是多样的（diverse)

最佳元模型可能因任务而异，本文提出的方法基于度量学习(metric-based)。

key idea是为FSL保留多种度量指标

元学习器（meta-learner)使用元训练任务上的任务聚类(task clustering)来选择和组合用于学习目标任务的多个度量。在元训练过程中，我们首先将元训练任务划分成簇(clusters)，使每个簇中的任务可能相关。在每个聚类中，训练一个深度嵌入函数作为度量，确保了公共指标尽在同一cluster的任务之间共享。此外，在元测试(meta-testing)期间，每一个目标FSL任务被分配给特定于任务的度量，该度量是由不同cluster定义的度量的线性组合。这样，多样的FSL任务可以从以前的学习经验中获得不同的度量标准。

主要任务是聚类，为了使任务聚类与深度网络兼容，以及能够处理具有不同数量标签的任务，提出了一种基于矩阵完成的任务聚类算法（matrix-completion based task clustering）。利用跨任务迁移性能衡量任务相似性，由矩阵S表示。S的(i,j)项是通过将第i个任务学习到的表示适应于第j个目标任务来估计的精度，依靠matrix comletion[1]来处理s中缺失和不可靠的项，最后应用spectral clustering（谱聚类）[2]生成任务分区。

Robust Task Clustering by Matrix Completion

算法首先评估应用单个任务模型（a single-task model）从模型i迁移到另一个任务j的效果,生成一个cross-task迁移效果的矩阵S。S cleaned and completed,产生一个用于谱聚类(spectral clustering)对称的任务相似度矩阵Y。

算法2：ROBUST TC:Robust Task Clustering based on Matrix Completion

Input: n个任务的集合$\mathcal {T} = {T{1},T{2},…,T_{n}}$

​ 任务簇clusters的数量K

Output: K个任务簇 $C_{1:K}$

1.单个任务模型的学习：为每个任务$T{i}$训练一个单任务模型$M{i}$

cross-task 迁移效果的评估

使用single-task model，adapt each $M{i}$ to each task $T{j}$(i!=j),得到一个性能分数$s_{i,j}$,形成了n x n 的成对的性能矩阵S。S通常是不对称的。

理想情况下，可以通过 在任务i上训练一个MNet并直接在任务j上进行评估 来估计迁移性能，但数据有限，单任务MNet的迁移性能比较低。因此，采用以下方法：

在任务i上训练一个CNN分类器（图一a所示）

then take only the encoder $M{i}^{enc}$ from $M{i}$ and freeze it to train a classifier on task j

产生了一个新的任务j模型，在$D{j}^{valid}$上进行验证，得到精度作为迁移性能$S{ij}$.表示了在任务i上学习到的特征表示如何适应任务j，从而表明任务之间的相似性。

Remark：Out-of-vocabulary problem

在文本分类任务中，将具有微调词嵌入的编码器从一个任务转移到另一个任务是困难的，因为这两个词汇表之间可能存在显著差异。因此，在学习单任务CNN分类器时，我们通常使用固定单词的嵌入。

Task Clustering Method

直接使用迁移性能进行任务聚类可能会存在效率和准确性问题，首先，对矩阵S中所有项（entries）的评估涉及进行 源-目标 迁移学习$O(n^2)$次，n是元训练任务数。当n>1000时，对矩阵的评估是不可接受的。其次，估计的cross-task性能(例如一些$S{ij}$或$S{ji}$scores通常是不可靠的，因为数据量小或标签噪声。当不确定值的数量较大时，它们会共同误导聚类算法输出错误的任务划分。针对上述问题，本文提出了一种基于矩阵补全（matrix completion）理论的任务聚类算法。具体来说，就是通过随机抽样任务对（task pairs)来处理大量项（entries)，对$S{ij}$和$S{ji}$进行评估,此外，我们通过保持$S{ij}$和$S{ji}$ score一致的 任务对(i, j) 来处理不可靠项和不对称问题。

首先，使用可靠的任务对生成部分观察(partially-observed)的相似矩阵y。具体来说，如果$S{ij}$和$S{ji}$足够高，那么任务{i, j}很可能属于同一个聚类并共享重要信息。相反，如果$S{ij}$和$S{ji}$足够低，那么它们往往属于不同的集群。为此，我们需要设计一种机制来确定性能是否足够高或足够低。由于不同任务的难度不同，固定的阈值是不合适的。因此，我们对目标迁移效果的均值和标准差定义了一个动态阈值。例如,$\mu{j} = mean(S{:j})$ and $\sigma{j}= std(S{:j})$,$S{:j}$是S的第j列。然后我们引入两个正参数p1and p2,将高低性能分别定义为高于$\mu{j}+p{1}\sigma_j{}$和低于$u{j} -p{2}\sigma{j}$。当$S{ij}$和$S{ji}$都足够高和足够低时，我们将它们的成对相似度分别设为1和0。其他任务对作为不确定任务对，标记为未观察任务对，不影响我们的聚类方法。生成了一个部分观察(partially-observed)的对称矩阵Y:

对话系统的用户意图分类

该数据集包含不同领域的人类用户和对话系统之间的对话记录，训练并为各种客户提供意图分类模型。在分类过程中，意图标签被分配给用户的话语(句子)。我们总共使用来自不同客户端的175个任务，并从它们中随机抽取10个任务作为我们的目标任务，对于每个元训练任务，我们随机抽取64%的数据到训练集，16%的数据到验证集，剩下的数据作为测试集。

这些任务的标签数量变化很大(从2个到100个）。为了适应FSL场景，对于目标任务，我们为每个标签保留一个示例(one-shot)，再加上20个随机挑选的标记示例来创建训练数据。

实验结果

在情感分类和对话意图分类上FSL的精度，情感分类的目标任务是5-shot任务，意图目标任务每类包含一个训练示例和20个随机标记的示例

结果：

  1. 利用先前从其他产品领域观察到的57个任务中学习到的知识，对12个few-shot产品情感分类任务进行了分析
  2. 通过利用165个先前从其他客户端数据中观察到的任务，10个few-shot对话框意图分类任务。

数据集有限，所有的监督学习baseline表现不佳。两种最先进的基于度量的FSL方法，匹配网络和原型网络，并不比其他基线表现得更好，因为单个度量不足以满足所有不同的任务。在任务更加多样化的意图分类上，所有的单度量或单模型方法(3-5)的表现都不如单任务CNN基线(1)。总的来说，所有单一训练任务模型的凸组合是表现最好的基线。

相比之下，我们的ROBUST TC-FSL与所有基线相比，始终结果更好。

Adaptive ROBUST TC-FSL

　　虽然ROBUSTTC-FSL在意图分类上比基线有所改善，但与情感分类相比，改善差距较小，因为意图分类任务在本质上更加多样化。在目标任务上的训练准确性也证明了这一点，其中几个任务无法找到任何可以提供适合其训练示例的指标的聚类。为了解决这一问题，我们提出了一种改进的算法来自动发现目标任务是否不属于任何一个任务簇。如果任务不属于任何clusters，它就不能从任何先前的知识中获益，因此就回到single-task CNN。当所有聚类对训练数据的准确率都不超过20%(在元验证任务中选择)时，该目标任务被视为“非聚类”。我们将该方法称为Adaptive ROBUSTTC-FSL，该方法在意图分类上比最佳ROBUSTTC-FSL结果提高了5%以上的性能。需要注意的是，自适应方法对情感任务没有影响，因为它们之间的关系更密切，所以重用聚类编码器总是比single-task CNNs获得更好的结果。

分析

  1. clusters数量的影响

    ROBUSTTC在情感分析(SA)上有5个聚类，在意图分类(intent)上有20个聚类。所有聚类结果明显优于single-metric baselines

  2. 聚类算法的影响

    ROBUSTTC是唯一能够聚类不同类别标签的算法

    即使在以往的任务聚类研究的所有二元分类任务(如情感分析任务)的设置下，我们的ROBUSTTC对于不同的FSL问题仍略有改善

结论

  • 提出了一种基于任务聚类的few-shot学习方法。该方法可以使用多个度量指标，当任务来自不同领域时，其性能明显优于以往的单度量方法。
  • 未来的工作包括将任务聚类思想应用于其他FSL算法，并探索线性组合之外更先进的聚类编码器组合方法
