Boosting Few-Shot Learning With Adaptive Margin Loss

Abstract

为了提高FSL的泛化能力，提出了两种自适应边界（adaptive margin）

  • class-relevant自适应边界：计算类别对之间的语义相似度，将特征映射空间中相似类别的样本分离
  • task-relevant自适应边界：将所有类别之间的语义上下文合并到一个抽样训练任务中，并开发了一个任务相关的自适应边界，以更好地区分来自不同类别的样本。

提出的方法可以提高当前基于度量的元学习方法的性能

Introduction

深度学习的成功

略

FSL简介

给定一组具有足够标记样本的基类和一组仅具有少量标记样本的新类，FSL的目标是通过从基类中学习一般知识来训练一个可以用于新类的分类器。

基于度量的FSL方法简介

学习一个很好的嵌入空间，来自同一个类的样本聚集在一起，而来自不同类的样本彼此相距很远，在嵌入空间内直接应用简单的距离度量，这些基于度量的方法的成功依赖于学习有区别的嵌入空间

本文在基于度量学习的小样本分类算法的基础上，通过在嵌入空间引入类别相关（class-relevant）或任务相关（task-relevant）的自适应边缘损失（Adaptive Margin Loss），改进了原有算法的分类效果。本文的核心观点是可以利用不同类别之间的语义相似性来生成自适应边界，具体地说就是让相似类别之间的边界类别比不相似类别之间的边界距离大。如下图所示：

希望增大“狗”和“狼”之间的边界距离，以增强相似类别的区分能力，提出一种自适应边缘损失的方法，利用损失函数去引导特征提取网络改进对于相似类别特征的表征能力。

该方法学习到了一个更具区分性和泛化能力的嵌入空间。

创新点
  • 在基于度量学习的小样本分类算法的基础上，引入了自适应边缘损失，改进分类效果
  • 提出两中自适应边缘损失的计算方式：CRAML和TRAML

Methodology

Metric-Based Meta-Learning

base class set CbaseC_{base}Cbase​ 包含nbasen_{base}nbase​个类别 每个类别都有充足的标签

novel class set CnovalC_{noval}Cnoval​ 包含nnovaln_{noval}nnoval​个novel class 每个novel class只有少量标签样本（比如少于5个）

FSL的目标就是获得一个在novel class上效果较好的一个分类器

Meta-learning 两个阶段：

  • meat-training：按照episode的方式训练模型，每个episode中，通过从base class set中随机采样来构造小的training set和test set，从而形成一个小的分类任务。用来更新模型
  • meta-testing：学习到的模型用来识别novel classes中的样本

基于度量的元学习通常假设存在一个嵌入空间，一个类中的样本聚集在class representations的周围，利用class representations来推断测试样本的标签

按照episode的方式进行训练

  • 每个episode中，从base class setCbaseC_{base}Cbase​中随机选取ntn_{t}nt​个类别用于episode的训练，记作CtC_{t}Ct​

普通的边缘损失函（naive additive margin loss，NAML）：

Lna=−1∣Q∣∑(x,y)∈Qlog⁡eD(F(x),ry)eD(F(x),ry)+∑k∈Ct\{y}eD(F(x),rk)+m\mathcal{L}^{\mathrm{na}}=-\frac{1}{|Q|} \sum_{(x, y) \in Q} \log \frac{e^{\mathcal{D}\left(\mathcal{F}(x), r_{y}\right)}}{e^{\mathcal{D}\left(\mathcal{F}(x), r_{y}\right)}+\sum_{k \in C_{t} \backslash\{y\}} e^{\mathcal{D}\left(\mathcal{F}(x), r_{k}\right)+m}} Lna=−∣Q∣1​(x,y)∈Q∑​logeD(F(x),ry​)+∑k∈Ct​\{y}​eD(F(x),rk​)+meD(F(x),ry​)​

它在类别两两之间加上了相同的边际 mmm，强迫不同类的样本之间分开一定的距离。这种简单加上等距离边际的方法在小样本测试集上（比如相似度很高的类别上）可能会带来错误。为了进一步精细化设计边际，我们借助类别之间的语义相似度，来自适应地生成边际。

Class-Relevant Additive Margin Loss

为了在特征嵌入空间中更好地分离相似类，两个类之间的边界应该是自适应的，即相似类的边界应该比不相似类的边界大，CRAML是根据类别间的语义相似性来调整边界

图片概述

在Meta-training阶段的每个episode中，首先从base class dataset中取样一个meta-training任务，meta-training任务中的类名输入到单词嵌入模型中，得到标签的语义向量，然后通过一个adaptive margin generator来得到类别对之间的边界，最后，将边际损失整合到分类损失中，从而获得自适应边际损失。通过最小化自适应边界损失来训练由嵌入模块和度量模块组成的元学习器；在Meta-test阶段，利用在训练阶段学习的嵌入模块和度量模块，使用简单的softmax来进行预测（不添加边界）

使用Glove单词嵌入模型来得到标签的语义向量，如上图所示，将一个类名，如wolf，输入到单词嵌入模型中，它会将类名嵌入到语义空间中，并返回一个语义单词向量。然后，我们构造一个类相关的边界生成器MMM。对于每对类，类iii和类jjj，MMM使用它们的语义词向量eie_{i}ei​和eje_{j}ej​作为输入，生成它们的边界:

通过适当地利用类之间的语义相似性，我们的class-relevant margin loss使得来自相似类的样本在嵌入空间中更可分离。更具区分性的嵌入空间将有助于更好地识别新类样本。

Task-Relevant Additive Margin Loss

到目前为止，我们都只考虑边际与任务无关。如果每次只考虑一个 meta-training task 中涉及的类别，那么可以更加精细地生成适合的边际。通过将一个 meta-training task 中的每个类与该 meta-training task 中其他类一一比对，我们可以衡量一个 task 内“相对的”语义相似度，从而生成适合这个 task 的边际。

具体来说，在一个meta-training task中，给定一个类别y∈Cty \in C_{t}y∈Ct​，生成器会根据类yyy和其他类Ct\{y}C_{t} \backslash \{y\}Ct​\{y}之间的语义相似度生成margin：

结论
  1. 使用adaptive margin loss训练模型，性能提升非常明显（Our Full Model vs. Original Classification Loss）
  2. 和Original Classification Loss相比，使用naive additive margin loss性能稍有改进，意味着在FSL中，仅在分类损失上增加一个固定的数字效果十分有限
  3. class-relevant margin additive loss对FSL的嵌入学习起到了作用（Class-Relevant Additive Margin Loss vs. Naive Additive Margin Loss)
  4. 通过考虑一个meta-training task中所有类别的信息，task-relevant additive margin loss训练的模型比用class-relevant margin additive loss训练的效果要更好

Conclusion

本文提出一种新的改进小样本学习的思路——边缘损失，本身在目标识别算法中边缘损失函数已经有一定的研究基础，而本文则是首次将其引入到小样本分类问题中。本文提出的两种边缘生成方法CRAML和TRAML均是从语义信息的角度出发，而边缘的计算还可以利用其他的信息或者约束。本文提出方法计算过程并不复杂，但却取得了较高的性能提升（PN提升超过10%，AM3提升2%-3%）。而唯一的遗憾是两种方法之间似乎是独立的，并没有相关联系，如何将两种方式结合，其实也是一种改进思路。

Q：训练时，用于测试的小样本新类并没有参与训练，那相似度是不是没法约束基类和小样本新类了？这样自适应边界不都是在基类和基类之间吗？

A：加自适应边际的目的是为了得到一个更好的且具有泛化能力的特征表示。由于在 meta-training 的时候，是按照一个个小样本 task（这些 task 都是从基类中抽取的）来更新模型的，所以，虽然自适应边际都是在基类和基类之间，但是不影响学到的好的特征表示泛化到新类。

Q：如何保证生成的语义相似度是准确的呢？

A：一，直接使用别人用海量语料预训练好的语言模型，准确度是足够的。二是，语义相似度其实只需要相对准确即可。
