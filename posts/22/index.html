Dynamic Memory Induction Networks for Few-Shot Text Classification

ACL 2020

Abstract

Dynamic Memory Induction Networks (DMIN)

　　该模型利用动态路由为memory-based的FSL提供更大的灵活性，以便更好地适应支持集，这是few shot分类模型需要具备的的重要能力。在此基础上，我们进一步开发了包含查询信息的归纳模型，旨在增强元学习的泛化能力。

Introduction

　　few-shot文本分类需要模型在标记样本有限的情况下进行分类，这对于许多应用来说很重要的，但仍然是一项具有挑战性的任务。早期关于FSL的研究采用数据增强和正则化技术来减轻数据稀疏导致的过拟合。最近的研究利用了元学习(Finn et al., 2017; Zhang et al., 2018;Sun et al., 2019) 来提取元任务之间可转移的知识。

few-shot文本分类的关键挑战：

  • 将支持集归纳为class-level representation (Gao et al., 2019)【1】。
  • 另一个突出的挑战是各种原因造成的instance-level多样性(Gao et al., 2019; Geng et al., 2019【2】)，导致很难找到一个类的固定原型 (Allen et al., 2019)【3】。

最近的研究表明，模型可以受益于query-aware 方法(Gao et al., 2019)。

　　在本文中，我们提出了DMIN来进一步应对上述挑战。DMIN利用动态路由，通过利用路由组件在训练期间和训练之后自动调整耦合系数的能力，为memory-based的FSL提供更大的灵活性，以便更好地适应支持集。在此基础上，我们进一步开发了带有 query information的归纳模型，从而在支持集中的不同实例中识别出与query更相关的样本向量。这两个模块是在DMIN中联合(jointly)学习的。

Dynamic Memory Induction Network

Overall Architecture

　　如图1所示，它建立在 two-stage few-shot 框架的基础上（ Gidaris　and Komodakis (2018)）。在监督学习阶段(绿色子图)，训练数据中的一部分类别被选作 base sets，它包含$C_{base}$个base classes，用于微调预训练句子编码器、训练分类器。

　　在元学习阶段(橙色子图)，我们构建一个“episode”来计算梯度，并在每次训练迭代中更新我们的模型。对于C-way K-shot问题，通过从训练集中随机选择C类，并在每个选择的类中选择K个样本作为支持集$S$，形成一个training episode。其余样本的子集用作查询集$Q$。用支持集训练模型，在查询集中最小化损失函数并更新模型。

Pre-trained Encoder

　　我们使用BERT-base进行句子编码，BERT是基于Transformer模型的多层双向Transformer编码器。

　　插入特殊分类嵌入([CLS])作为第一个token，并添加special token ([SEP])作为最终token。我们使用从[CLS]输出的d维隐藏向量作为给定文本 $x$ 的表示 $e$ ：$e = E(x|\theta)$。预训练的BERT模型提供了强大的上下文相关句子表示，可用于各种目标任务，适用于few-shot文本分类任务 (Bao et al., 2019)【1】。

　　我们在监督学习阶段微调预训练的BERT编码器。对于每个输入文档$x$，编码器$E(x|\theta)$(带有参数$\theta$)将输出一个d维的向量$e$。$W_{base}$是一个矩阵，为每个base class保存一个class-level 向量，作为元学习的 base memory 。$E(x|\theta)$和$W_{base}$将在元训练过程中进一步调整。实验表明，将先前的模型替换为预训练编码器后的性能优于最先进的模型。

Dynamic Memory Module

　　在元学习阶段，为了从给定的支持集中归纳出类级表示，我们通过memory matrix $W_{base}$，根据监督学习阶段学习到的知识，开发了一个动态记忆模块(DMM)。与静态内存不同，DMM利用动态路由为从base classes中学习的memory提供更大的灵活性，以更好地适应支持集。路由组件可以在训练期间和训练之后自动调整耦合系数，这本质上适合FSL的需要。

　　具体来说，支持集中的样本首先由BERT编码成样本向量$\{e_{c,s}\}_{s=1}^{K} $，然后被馈送到后面的 dynamic memory routing process。

Query-enhanced Induction Module

　　在样本向量$\{e_{c,s}^{\prime}\}_{s = 1,…,K}$ 适应并且查询向量$\{e_{q}\}_{q=1}$被预训练编码器编码之后，现在合并queries来创建一个Query-guided Induction Module (QIM)。目的是在支持集的(adapted)样本向量中识别与query更相关的向量，以便构建类级向量来更好地对query进行分类。由于动态路由可以自动调整耦合系数来帮助增强相关的queries和样本向量，并且penalizes不相关的queries和向量，因此QIM通过将adapted sample vectors视为关于新类的背景知识的memory来重用DMR过程，并且从与查询更相关/相似的adapted sample vectors中归纳类级表示。

Experiments

Dataset and Evaluation Metrics

数据集：miniRCV1 ODIC

评价指标：分类精度

　　分别从miniRCV1 and ODIC的测试集中随机生成100和300个meta-episodes。在1-shot和5-shot场景下，为每个episode中的每个类取10个测试文本用于评估。

Implementation Details

　　使用谷歌预训练的BERT-Base模型作为文本分类器，在训练过程中微调模型。在ODIC and miniRCV1上，base classes的数量$C_{base}$分别设置为100和20。DMR迭代的次数为3。用$C = [5,10]$和$K = [1,5]$来构建基于episode的元训练。在每个training episode中，除了使用K个样本文本作为支持集之外，还使用10个查询样本作为查询集。比如，对于5-way 5-shot场景，一个training episode中有$10 \times 5+ 5 \times 5$个文本。

Overall Performance

　　模型的准确度和标准偏差如表1和表2所示。我们可以看到，DMIN始终优于所有现有模型，并在两个数据集上都取得了新的最先进的结果

　　LwoF构建了一个两阶段的训练过程，记忆模块从监督学习中学习并用于元学习阶段，但训练后记忆机制是静态的，而DMIN使用动态记忆路由来自动调整训练后的耦合系数，以推广到新的类别，并显著优于LwoF。还要注意一些基线模型的性能比原论文中报告的要高（如关系网络和归纳网络），这是因为我们使用BERT代替了基于BiLSTM的编码器。BERT编码器通过强大的上下文表示能力改进了基线模型，我们的模型可以通过dynamic memory routing方法进一步优于这些模型。即使有这些更强的基线，提出的DMIN在两个数据集上都始终优于它们。

Ablation Study

　　表3中分析了DMIN的不同组成部分对性能的影响。具体来说，我们移除了DMM和QIM，并改变了DMR迭代的次数。我们看到最佳性能是通过3次迭代实现的。实验结果表明了动态记忆模块和带有查询信息的归纳模块的有效性。

Further Analysis

　　图2是在10-way 5-shot设置下，应用DMM前后支持样本向量的t-SNE可视化。我们随机选择一个包含50个文本(每类10个文本)的支持集，获得DMM前后的样本向量。我们可以看到，DMM产生的支持向量被更好地分离，证明了DMM在利用监督学习经验来编码样本特征和类特征之间的语义关系以进行few-shot文本分类方面的有效性。

Conclusion

　　Dynamic Memory Induction Networks建立在具有动态路由的外部工作记忆的基础上，利用动态路由来寻找先前的学习经验，利用外部记忆来更好地适应支持集，从而泛化到unseen类。
