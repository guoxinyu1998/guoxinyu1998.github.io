Edge-Labeling Graph Neural Network for Few-Shot Learning

Abstract

  • 提出了edge-labeling graph neural network (EGNN)，在edge-labeling graph上采用深度神经网络用于FSL。

  • 以前用于FSL的GNN方法都是建立在node-labeling框架上的，对类内相似性和类间的不相似性进行隐式的建模。

  • 相比之下，EGNN学习预测图中的edge-labels，而不是node-labels，能够通过直接利用intra-cluster similarity和inter-cluster dissimilarity，实现显式聚类的演化。

  • EGNN参数训练：EpisodeEpisodeEpisode，an edge-labeling loss ，得到可适用于少量数据的模型。

  • EGNN显著提升了分类性能。

Introduction

  • 元学习在FSL方面的应用，例如MN、PN、RN、MAML之类的……以及FSL的目的

  • GNN可以利用神经网络处理数据间复杂的的关系，列举了一部分论文……

    • GNNs通过消息传递（message passing）迭代的完成和邻居(neighbors)节点间的数据聚合，从而可以表示数据间复杂的关系。
    • FSL中需要充分利用Support set和Query set之间的关系，列举了一部分论文如MN、PN
    • GNN很适合解决FSL问题

  • 前人用GNN解决FSL的工作：

    • Few-shot learning with graph neural networks首先提出构建一个图，其中支持集中所有样本和一个query紧密相连，即构建一个从support set 到query的全连接图 。每个输入node由嵌入特征和给定的标签信息（One-hot编码）表示。然后，通过聚合邻居node来更新节点feature，完成对query的分类。
    • Transductive propagation network for few-shot learning(TPN) 提出了TPN，在通过深度神经网络得到的节点特征上进行传导传播。在测试时，它使用一个公共的Graph参数集在整个支持和查询实例上迭代地传播One-hot编码标签。
    • 上述 方法主要基于node-labeling框架，隐式的建模类内的相似性和类间的不相似性

  • edge-labeling 模型

    • 显式聚类，结合表征学习和度量学习
    • 不需要预先指定类的数目（例如，class-way）,而node-labeling框架需要

  • EGNN

    • 由多个层组成，每个层由node更新块和edge更新块组成。具体而言，跨层的EGNN不仅更新node特征，而且明确调整edge特征，这些edge特征反映了两个连接node对的edge-label，并直接利用了类内相似性和类间不相似性
    • 如下图所示，在进行多次node 特征和edge特征更新后，可以从最终的edge特征获得edge−labeledge-labeledge−label预测
    • 计算edge loss，更新模型参数，Episode训练策略
    • EGNN可以实现对所有query数据整体的转导式传播（转导式传播见后文），一次就可以对所有的query 进行预测。可以证明它在只有少量样本的情况下能够做出更具鲁棒性的预测。
    • 可以处理各种数量的类别，不需要remodeling or retraining

  • 贡献：

    • 提出EGNN，利用类内相似性和类间不相似性迭代更新edge−labeledge-labeledge−label，该模型能适用于各种数量的类，不需要retraining
    • EGNN包含若干个层，每个层包含a node-update block and an edge-update block，在Episode训练模式下计算参数
    • 在两个图像分类benchmark上的效果显著提高，进行了消融实验，证明了显式聚类(explicit clustering)和使用intra-cluster similarity and inter-cluster dissimilarity 的优势。

实验结果不多说了，看图，值得注意的是：

  • EGNN比node-label GNN效果更好

  • 转导的方法比非转导方法效果更好

在TPN中，只有支持集标签传播到query，因此query仅通过他们的嵌入特征的相似性进行传播；相反，EGNN在query样本之间考虑了更复杂的信息交互方式，不仅将它们的Node特征相互传播，还传播edge-label信息

TPN中node特征是固定的，而EGNN中node和edge特征都是动态变化的

消融研究

  1. EGNN网络的层数
  2. EGNN在Node和Edge更新中分别利用了类内相似性和类间不相似性，为了验证这种方法的有效性，进行了仅使用类内聚合（仅适用类内相似度，忽略类间不相似度）的实验，并将结果与使用两种聚合获得的结果进行了比较

  1. 与基于Node的GNN相比，EGNN更能适用于任意场景下的元测试，尤其是用于元测试和元训练的类别数量不同时。进行了交叉实验，如下表所示：

  • 模型在5-way 5-shot设置下训练的，在10-way 5-shot设置下进行测试，可以观察到EGNN可以成功地扩展到不同的小样本场景中，而无需重新训练模型
  • 而基于Node的GNN在更换测试场景后，模型甚至不可用，模型的容量和参数取决于类别的数量。
  1. 对node-label GNN和EGNN的node特征进行了t-SNE可视化

上面是GNN，下面是EGNN，从左到右依次是嵌入特征、第一层、第二层、第三层，x表示的是query样本，o表示的是support样本

  • 在经过第一层传播之后，GNN中支持集的聚合效果更好（分布在四周的聚类），然而，与此同时query样本也高度聚集（中心的聚类），在经过网络的传播后，query样本并没有和对应类的support样本靠近
  • 而EGNN中，随着传播的进行，如果query和support样本的标签不同，他们之间的距离就会变大，而相同的标签的query和support样本会靠近。

注：或许另一篇Paper中将EGNN结合原型网络来做文本分类的灵感来源于此吧：

  • 归纳网络那篇论文中直接将GNN用于ARSC，但是效果并不好
  • 相比于GNN，EGNN学习到的node特征在空间中的分布更好（query与不同类的support样本距离大，与相同类的support样本距离小），所以那篇文章的作者将EGNN学习到的Node特征嵌入到原型网络中进行度量，在ARSC上效果很好

分析：

  • N-way K-shot
    • 5-way 3-shot，每类3个query样本，总共15个support样本，15个query样本，体现在图中就是，横向和纵向的小方格数都为30，前15个表示support，后15个表示query，小方格就表示了node之间的eij1e_{ij1}eij1​特征（说白了就是相似度）
  • eij1e_{ij1}eij1​的初始化：
    • 明确eij1e_{ij1}eij1​的定义规则
    • 先看对角线，对角线表示某个node和它自身的相似度，不论是support node还是query node，肯定都为1（红色）……
    • 接着看支持集node，因为初始化时support node的标签是已知的，所以可以判断两个node标签是否相同，相同则edge=1；不同为0（蓝色部分）
    • 再看query node，query node标签未知，也就没办法判断它和其他nodes是否属于同一类，只能判断它和自身是同一类，所以对角线为1，其他edges全部初始化为0.5（灰色部分）
    • 经过图神经网络的传播后，edge feature 逐渐演变为 ground-truth edge label
